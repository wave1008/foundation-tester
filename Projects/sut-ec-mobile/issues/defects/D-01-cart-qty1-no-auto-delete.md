# D-01 カート: 数量1で「−」しても明細が削除されない(TC-42 不一致)

| 項目 | 内容 |
|---|---|
| 対象 | カート / 数量ステッパー |
| 対応 SC/TC | SC-42 / TC-42 |
| 重大度 | 中(代替手段=ゴミ箱削除あり) |
| 状態 | 起票 |
| 検出 | 自動シナリオ `カートの数量を操作できること.S0020` / iPhone 17 Pro・iOS 27.0 |
| 起票日 | 2026-07-20 |

**再現手順**
1. 任意の商品をカートに1点入れる(数量1)。
2. カート明細の「−(数量を減らす)」をタップする。

**期待結果(TC-42)**: 数量が1→0になり、その明細がカートから自動削除される。

**実際の挙動**: 数量1のとき `#btn_qty_decrement`(数量を減らす)が **disabled(無効)** で、タップしても
削除されず数量1のまま残る。明細の削除は右上のゴミ箱(`#btn_remove_<productId>`)でのみ可能。

**証拠**: カート明細の ft_snapshot に `Button "数量を減らす" id=btn_qty_decrement disabled`(数量=1)。
シナリオ `カートの数量を操作できること.S0020` は TC-42 準拠で「−後にカートが空」を期待するため **RED**
(仕様違反を検出。scene3 後始末で残留は消すため副作用なし)。

**仕様裁定(要判断)**: 次のいずれか。
- (A) 実装を TC-42 に合わせ「数量1で−→明細を自動削除」にする(テストは修正で緑化)。
- (B) TC-42 を実装に合わせ「数量1で−は無効、削除はゴミ箱」に改訂する(UX 的には妥当な選択肢)。
裁定まではテスト RED を維持し、本ファイルの状態で追跡する。
