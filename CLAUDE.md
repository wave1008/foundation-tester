# foundation-tester

## コメント規約

コメントは「現行コードの理解に必要な事実・制約・設計理由」のみを書く。

書いてよいもの:
- 現行の制約・不変条件・落とし穴(コードから読み取れないもの)
- 「なぜこうなっているか」の設計理由(現在形で)
- 実行時の状態説明(「キューが空だったので即実行される」等は履歴ではない)

書かないもの(経緯は git log と docs/ に残す):
- 移動元・抽出元の記録(「〜から移動した」「〜節から抽出した」)
- 旧仕様との比較(「元々〜だったが」「旧実装は〜」「以前は〜していた」)
- 日付・指示・要件番号のタグ(「(2026-07-11 指示)」「(要件1)」)
- リファクタ時の無変更宣言(「ロジックは変更していない」)
- 計画のフェーズ番号(「Phase 3 で統一」)。チューニング経緯が必要な箇所は docs/performance-tuning.md を参照する
